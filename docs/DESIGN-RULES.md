# Development Rules

## Scope Control

Only modify the files requested.

Do not refactor unrelated code.

Do not redesign completed sections.

Leave all other components untouched.

---

## Component Preservation

If a section has already been approved:

Do not redesign it.

Only improve spacing, responsiveness, accessibility, or performance.

---

## Design Fidelity

When a reference image exists inside reference-assets:

Treat that image as the source of truth.

Implement it faithfully.

Do not reinterpret the design.

---

## Asset Management

Do not replace approved project images.

Do not replace approved branding.

Do not introduce placeholder imagery unless requested.

---

## Copilot Behavior

When making edits:

Make the smallest change necessary.

Prefer editing existing code over rewriting entire components.

Avoid generating new layouts unless specifically requested.
